
## 📖 功能說明

### 1. 搜尋功能（真實 API 資料）

**步驟**：
1. 在搜尋框輸入 User ID（純數字，例如：12345）
2. 點擊「🔍 查詢」按鈕
3. 系統會：
   - 從 BitoPro API 載入資料
   - 訓練風險評估模型（首次查詢）
   - 分析該用戶的風險分數
   - 顯示詳細特徵和風控建議

**顯示內容**：
- 用戶基本資訊（User ID、風險分數、風險等級）
- 交易統計（交易筆數、交易總額、買入比例等）
- 轉帳行為（加密貨幣轉帳、TWD 轉帳、USDT 兌換等）
- 風險評估（判定結果、信心度、KYC Level 2 狀態）
- 詳細特徵分析（所有特徵的數值）
- 風控建議（根據風險等級提供不同的處置建議）

### 2. 總覽功能（模擬資料）

**步驟**：
1. 點擊「📊 總覽」按鈕
2. 系統會：
   - 生成 100 個模擬帳號
   - 顯示整體風險分布
   - 列出可疑帳號 TOP 10

**顯示內容**：
- 5 個 KPI 卡片（總帳號、可疑帳號、高風險、平均風險、正常帳號）
- 風險分數分布圖
- 特徵重要性圖
- 可疑帳號列表
- 帳號詳細分析

## ⚙️ 系統架構

### 資料來源
- **真實資料**：BitoPro API（搜尋功能）
  - `usdt_twd_trading`：USDT/TWD 交易資料
  - `crypto_transfer`：加密貨幣轉帳資料
  - `user_info`：用戶資訊
  - `twd_transfer`：TWD 轉帳資料
  - `usdt_swap`：USDT 兌換資料
  - `train_label`：訓練標籤

- **模擬資料**：內建生成器（總覽功能）
  - 85% 正常帳號
  - 15% 異常帳號

### 特徵工程
系統會從原始資料中提取以下特徵：
- 交易相關：交易筆數、交易總額、平均交易額、買入比例、市價單比例
- 轉帳相關：加密貨幣轉帳次數、TWD 轉帳次數、USDT 兌換次數
- 用戶相關：年齡、性別、KYC Level 2 狀態
- 行為相關：幣種數量、交易標準差等

### 風險評估模型
- **演算法**：Random Forest Classifier
- **參數**：
  - n_estimators=100
  - max_depth=10
  - min_samples_split=10
  - class_weight='balanced'
- **輸出**：風險分數（0-1）和判定結果（Normal/Suspicious）

### 風險等級
- **高風險**：風險分數 ≥ 0.7（紅色）
- **中風險**：0.5 ≤ 風險分數 < 0.7（黃色）
- **低風險**：風險分數 < 0.5（綠色）

## 🔧 故障排除

### 問題 1：API 載入失敗
**症狀**：顯示「⚠️ API 暫時無法使用」

**解決方案**：
1. 檢查網路連線
2. 等待幾分鐘後重試
3. 如果持續失敗，使用「總覽」功能查看模擬資料

### 問題 2：找不到用戶
**症狀**：顯示「⚠️ 找不到用戶」

**原因**：
- 輸入的 User ID 不存在於 BitoPro 資料中
- User ID 格式錯誤（必須是純數字）

**解決方案**：
1. 確認輸入的是純數字
2. 嘗試其他 User ID
3. 使用「總覽」功能查看可用的帳號

### 問題 3：模型訓練緩慢
**症狀**：首次查詢需要等待較長時間

**原因**：
- 首次查詢需要訓練模型
- 訓練資料量較大

**解決方案**：
- 耐心等待（通常 30-60 秒）
- 訓練完成後，後續查詢會很快

### 問題 4：IndexError
**症狀**：顯示「index 1 is out of bounds」錯誤

**狀態**：✅ 已修復

**修復內容**：
- 處理單一類別情況
- 自動補充模擬資料
- 添加完整的異常處理
