# BitoGuard Demo 演示指南

## 🎯 演示目標

展示 BitoGuard 如何使用真實 BitoPro API 資料進行即時風險評估。

## 📋 演示準備

### 1. 環境檢查

```bash
# 確認依賴已安裝
pip install -r requirements.txt

# 測試 API 連線
python test_real_api.py
```

預期看到所有端點都顯示「✓ 成功」。

### 2. 啟動 Demo

```bash
# Windows
run_real_api_demo.bat

# Linux/Mac
./run_real_api_demo.sh
```

瀏覽器會自動開啟 `http://localhost:8501`

## 🎬 演示腳本

### 第一部分：介紹系統（1 分鐘）

**說明重點：**
1. BitoGuard 是一個 AI 驅動的加密貨幣風險評估系統
2. 整合真實 BitoPro API 資料
3. 使用機器學習模型進行即時風險評分
4. 參考台北城市儀表板的專業設計

**展示畫面：**
- 指出頂部的「LIVE」指示器（表示即時系統）
- 展示搜尋框和總覽按鈕
- 說明深色主題和緊湊卡片設計

### 第二部分：查詢真實用戶（3 分鐘）

**操作步驟：**

1. **輸入 User ID**
   ```
   在搜尋框輸入：12345
   點擊「🔍 查詢」按鈕
   ```

2. **等待載入**（首次約 30-60 秒）
   - 說明：「系統正在從 BitoPro API 載入資料並訓練模型」
   - 強調：「這只需要一次，後續查詢會很快」

3. **解讀結果**

   **用戶資訊卡片：**
   - User ID
   - 風險分數（0-1）
   - 風險等級（低/中/高）
   
   **交易統計：**
   - 交易筆數：顯示用戶的交易活躍度
   - 交易總額：總交易金額
   - 平均交易：每筆交易的平均金額
   - 買入比例：買入 vs 賣出的比例
   
   **轉帳行為：**
   - 加密貨幣轉帳次數
   - TWD 轉帳次數
   - USDT 兌換次數
   - 使用的幣種數量
   
   **風險評估：**
   - AI 模型判定結果
   - 信心度百分比
   - KYC Level 2 狀態

4. **查看詳細特徵**
   - 向下滾動查看「詳細特徵分析」表格
   - 說明：「這些是從 6 個不同 API 端點提取的 18+ 個特徵」

5. **風控建議**
   - 根據風險等級顯示不同的處理建議
   - 高風險：立即凍結、身份驗證
   - 中風險：加入監控名單
   - 低風險：維持常規監控

### 第三部分：查詢其他用戶（2 分鐘）

**操作步驟：**

1. **查詢另一個用戶**
   ```
   輸入：67890
   點擊「🔍 查詢」
   ```

2. **對比結果**
   - 注意：這次查詢很快（2-5 秒）
   - 說明：「因為資料已經快取，模型已經訓練好」

3. **展示不同風險等級**
   - 如果是低風險：綠色指示器
   - 如果是中風險：黃色指示器
   - 如果是高風險：紅色指示器

### 第四部分：總覽模式（2 分鐘）

**操作步驟：**

1. **點擊「📊 總覽」按鈕**

2. **展示 KPI 卡片**
   - 總帳號數
   - 可疑帳號數
   - 高風險帳號數
   - 平均風險分數
   - 正常帳號數

3. **展示圖表**
   - 風險分數分布圖：顯示正常 vs 異常的分布
   - 特徵重要性圖：哪些特徵對風險評估最重要

4. **展示可疑帳號列表**
   - TOP 10 可疑帳號
   - 可以點擊選擇查看詳細分析

### 第五部分：技術亮點（2 分鐘）

**說明重點：**

1. **真實 API 整合**
   - 從 6 個 BitoPro API 端點獲取資料
   - 資料快取機制（1 小時）
   - 錯誤處理和超時控制

2. **機器學習模型**
   - Random Forest 分類器
   - 18+ 個特徵
   - 自動特徵工程
   - StandardScaler 標準化

3. **效能優化**
   - 資料快取（減少 API 請求）
   - 模型快取（避免重複訓練）
   - 平行處理（加快訓練速度）

4. **UI/UX 設計**
   - 台北城市儀表板風格
   - 深色主題、緊湊卡片
   - 即時載入指示器
   - 清晰的風險視覺化

## 📝 常見問題

**Q: 為什麼首次查詢這麼慢？**
A: 需要從 API 載入資料並訓練模型，只需要一次。

**Q: 資料多久更新？**
A: API 資料快取 1 小時，可以重新啟動來清除快取。

**Q: 可以查詢任何 User ID 嗎？**
A: 可以，但只有在 BitoPro 系統中存在的用戶才會有資料。

**Q: 風險分數是如何計算的？**
A: 使用 Random Forest 模型，基於 18+ 個特徵進行預測。

**Q: 可以匯出報告嗎？**
A: 目前版本還不支援，這是未來的改進方向。
